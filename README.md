# Handwritten Digits and Operators Classifier

Данный проект посвящен разработке и анализу архитектур сверточных нейронных сетей (CNN) для распознавания рукописных цифр и математических операторов (16 классов). Проект охватывает полный цикл ML-разработки: от подготовки данных и глубокого исследования влияния гиперпараметров на качество модели до реализации пайплайна инференса для пользовательских изображений.

## 📊 Данные и предобработка (Preprocessing)

В качестве исходных данных используется датасет, содержащий 80 000 изображений размерностью 32x32 пикселя (разбиение: 64 000 тренировочная выборка / 16 000 тестовая). В наборе представлены цифры от 0 до 9, а также символы `+`, `-`, `*`, `/`, `[`, `]`. 

Нейросеть обучалась на изображениях, представляющих собой черные символы на белом фоне.

Для обеспечения устойчивости модели при инференсе на реальных пользовательских рисунках был разработан кастомный пайплайн предобработки с использованием `OpenCV`:
* Инверсия и создание маски для нахождения `bounding box` символа.
* Кадрирование с добавлением padding'а (15% от размера) для корректного центрирования.

## 🏗 Эксперименты и архитектуры (Model Analysis)

Ключевой задачей проекта было исследование влияния различных архитектурных решений на итоговую точность модели. В качестве бейзлайна (Base Model) была выбрана стандартная CNN: `Conv2D(32) -> MaxPool -> Conv2D(64) -> MaxPool -> Flatten -> Dense(128) -> Dense(16)`. 

Отталкиваясь от бейзлайна, были протестированы следующие гипотезы:

1.  **Размер фильтров (Kernel Size):** Переход от 3x3 к 5x5 и 7x7. Более крупные фильтры на ранних слоях позволили сети лучше улавливать глобальные очертания символов (особенно крупных, таких как `[`, `]`, `+`), что немного повысило точность (до 98.58%).
2.  **Глубина сети (Stacking Convolutions):** Чередование двух сверточных слоев перед пулингом (`Conv-Conv-Pool`). Это позволило увеличить эффективное рецептивное поле при сохранении большего количества нелинейностей (VGG-паттерн). Эта гипотеза оказалась **самой успешной**.
3.  **Способ уменьшения размерности:** Использование `strides=2` вместо `MaxPooling2D`. Модель потеряла инвариантность к небольшим сдвигам, что привело к просадке качества.
4.  **Емкость модели (Model Capacity):** Увеличение количества фильтров (64 и 128) и их уменьшение (8 и 16). Сужение сети привело к недообучению (97.72%), в то время как расширение не дало значимого прироста по сравнению с чередованием слоев, показав избыточность параметров для данной задачи.
5.  **Регуляризация:** Добавление `BatchNormalization` и `Dropout(0.5)`. Для данной размерности данных сильная регуляризация оказалась излишней и не улучшила базовые метрики.
